# V1.3 代码级执行清单 (Execution Checklist)

## ✅ V1.3 完成标准 (Definition of Done)

只有同时满足以下 5 条，V1.3 版本才算真正完成：

1. **V1.3 MVP 范围内，任意一笔常规策略订单都能通过 `chain_id` 完整追溯**：`signal → risk → execution → order → trade`。
2. 所有 MVP 范围内的订单来源统一通过 Pipeline 入口处理。
3. `actual_pos` 的变动与 `TradeData` 的增量回调完全一致。
4. 必须跑通两条自动化测试验证路径：**正常成交路径 (PASS)** 与 **风控阻断路径 (REJECT)**。
5. **API 零修改兼容**：项目原有历史策略文件无需修改任何一行代码即可直接运行。

## 🚫 边界控制 (Non-Goals)

* ❌ 止损单 (`StopOrder`) 与换月单 (`Rollover`) 管道接入（延至 Iteration 2/3）。
* ❌ 复杂业务风控规则（仅跑通纯 PASS 或手动抛出 REJECT 测试链路）。
* ❌ 智能拆单逻辑（1 Signal = 1 Exec）。
