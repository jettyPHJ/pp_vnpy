# 📈 系统版本与能力演进体系 (Versioning & Roadmap)

> **Purpose:** 将系统开发阶段与“能力边界”严格绑定。每个版本必须定义解决的核心问题及提供的系统能力。

## 🧱 核心演进路线 (Baseline -> Live Trading)

| 版本           | 阶段名称                 | 核心解决问题 / 能力解锁                                      | 架构映射层级           |
| :------------- | :----------------------- | :----------------------------------------------------------- | :--------------------- |
| **V1.0** | 基线回测                 | 跑通基础 PnL 输出与投研闭环                                  | Research               |
| **V1.1** | 连续合约正确性           | 解决换月价格跳空与指标失真                                   | Data                   |
| **V1.2** | 换月成本建模             | 引入真实的换月损耗模拟与换月审计记录                         | Data / Execution / Ops |
| **V1.3** | **意图到成交追踪** | **引入 TraceID，实现从逻辑信号到物理成交的闭环与审计** | Strategy / Ops         |
| **V1.4** | **执行摩擦真实化** | **模块化滑点与费率模型，支持流动性冲击成本模拟**       | Execution              |
| **V2.0** | 实盘执行分离             | 将回测执行与实盘执行抽象为统一接口*(如 QMT/CTP 适配)*      | Execution / Risk       |
| **V2.1** | 机构级对账与运维         | 建立实盘断线脏数据清洗与 Broker 快照自动校准防线             | Ops                    |
| **V3.0** | 组合与智能控制           | 多策略权重分配、市场状态自适应调仓                           | Portfolio              |
