# 📈 极简量化交易系统演进路线图 (System Evolution Roadmap)

## 🎯 核心架构理念

1. **研究与实盘同源**：底层撮合、换月逻辑必须与实盘规则绝对一致。
2. **信号与风控物理隔离**：`ctastrategy` 只管“什么时候该买卖”，`riskmanager` 决定“能不能买、买多少”。
3. **按需扩展，绝不贪大**：不要一次性引入所有 vn.py 官方模块，按照实盘最短路径逐个击破。
4. **验证驱动开发 (VDD)**：每引入一个新模块，必须通过双账本审计和历史/实盘数据对账。

---

## 🗺️ 分阶段演进路径

### Phase 1: 夯实投研底座（📍 当前阶段）

**目标**：打造一个绝对准确、可信赖的 CTA 单品种回测引擎。

* **主战场**：`vnpy_ctastrategy` (引擎内核) & `vnpy_ctabacktester` (评测端)
* **核心任务**：
  * [X] 彻底解决连续合约与物理合约的时序对齐与坐标转换。
  * [X] 完善换月摩擦费用的双轨制财务对账（Daily vs Rollover Logs）。
  * [X] 建立完善的错误拦截机制（缺失 K 线阻断、发单失败拦截）。
  * [ ] 稳定跑通各类策略的参数优化（遗传算法/穷举），确保 `warmup_days` 等参数无损透传。

### Phase 2: 风控层前置与隔离（🚀 下一阶段重点）

**目标**：将仓位管理和风控逻辑从策略代码中剥离，建立独立的拦截层。

* **新增目录**：新建并重度二开 `vnpy_riskmanager/`
* **核心任务**：
  * **流控与熔断**：单日发单限制、撤单上限、最大净值回撤阻断。
  * **动态暴露度控制**：全局仓位上限硬约束。
  * **LLM 风控接口储备**：为将来接入大模型评估“市场微观结构风险”或“政策面风险”预留统一的拦截器 API。

### Phase 3: 实盘交易通道接入

**目标**：将系统发出的信号转化为真实的券商/期货公司委托单。

* **新增目录**：按需新建 `vnpy_qmt/` 或 `vnpy_ctp/` (根据实际使用的券商通道决定是否需要独立目录，若改动小则直接复用官方库)。
* **核心任务**：
  * 解决断线重连、委托回报延迟等网络级异常。
  * 确保实盘账户真实持仓与策略内部逻辑持仓的每日自动对账与同步。

### Phase 4: 本地数据资产沉淀

**目标**：摆脱对外部回测数据源的单纯依赖，建立实盘伴生的高质量数据集。

* **引入模块**：`vnpy_datarecorder` & `数据库组件 (MySQL/SQLite)`
* **核心任务**：
  * 盘中静默录制真实的 Tick 数据和合成 1 分钟 Bar。
  * 为未来的深度学习（Transformer/GNN）因子挖掘积累无损的、带有真实盘口特征的训练集。

### Phase 5: 截面与组合扩展（🔮 远期规划）

**目标**：从单品种时间序列 CTA，升级为多品种/截面因子协同调仓的组合系统。

* **引入模块**：`vnpy_portfoliostrategy`
* **核心任务**：
  * 支持多合约等权/波动率平价调仓。
  * 支持跨品种状态判定（如螺纹钢/铁矿石比价驱动的交易）。

## 📂 最终目标目录树架构示意

```text
root/
│
├── vnpy_ctastrategy/       # 【投研核心】CTA回测内核、信号生成、换月与撮合
├── vnpy_ctabacktester/     # 【界面组件】回测 UI、报告生成、参数优化工具
├── vnpy_riskmanager/       # 【风控大脑】前置阻断、暴露度约束、LLM风控预留
├── vnpy_qmt/               # 【通道接入】特定极速交易终端的底层 API 封装及稳定性补丁
│
├── _test_vx/               # 【实验与验真】回归测试、财务对账脚本、最小复现代码
├── run.py                  # 【单测启动】回测与优化的总入口
├── run_trading.py          # 【实盘启动】集成 Gateway、RiskManager 和 Strategy 的生产环境入口
```
